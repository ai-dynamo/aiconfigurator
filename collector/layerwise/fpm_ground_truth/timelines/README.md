## Normalization / inflation (the key caveat)

**Golden â€” de-spun + scaled to FPM.** Under nsys the multi-process serve's all-reduce
(`cross_device_reduce_2stage`) **busy-waits** for cross-rank sync â€” a probe artifact that does
NOT happen in real serving (~23 ms/step captured, ~25 ms eager of pure spin). Each all-reduce
instance is therefore **capped at the real transfer floor = 22.4 Âµs/call** (the global captured
p25; AR data volume is tiny and size-independent). The step is then compacted and its remaining
GPU-idle **scaled so the step span equals golden's real FPM latency** (captured 16.0 ms, eager
40.6 ms). Kernel busy durations are kept real (CUPTI on-GPU exec is accurate); only idle is
scaled. The residual idle is the genuine host-dispatch/launch/sync overhead serving pays.

**Collector â€” RAW nsys scale (factor 1.0), no comm (single GPU).** Its large inter-kernel idle
IS the diagnostic (isolated host-dispatch starvation); the raw nsys span already â‰ˆ the
collector's reported isolated over-prediction, so it is not de-inflated.
